# Contributing to Isma-L-

Thank you for your interest in contributing to our project! We appreciate your help and are excited to hear your ideas. Below are guidelines to help you get started:

## Getting Started
1. **Fork the Repository**: Click the 'Fork' button in the top-right corner to create a copy of the repository in your own GitHub account.
2. **Clone the Repository**: Clone your fork to your local machine using:
   ```bash
   git clone https://github.com/YOUR-USERNAME/Isma-L-.git
   ```
3. **Create a New Branch**: Itâ€™s a good idea to create a new branch for your feature or bug fix:
   ```bash
   git checkout -b my-feature-branch
   ```

## Making Changes
- Make your changes in the codebase and be sure to test them thoroughly.
- Follow our coding conventions and ensure code quality.

## Committing Changes
- Make your commits descriptive: `git commit -m "Add a new feature"`
- Squash your commits if they are related.

## Push Your Changes
Push your changes to your forked repository:
```bash
git push origin my-feature-branch
```

## Creating a Pull Request
1. Go to the original repository where you want to propose your changes.
2. Click on the `Pull Requests` tab.
3. Click the `New Pull Request` button.
4. Choose your branch and submit your PR.

## Code of Conduct
Please adhere to our [Code of Conduct](CODE_OF_CONDUCT.md) while participating in this project.
